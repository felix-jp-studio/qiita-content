TypeScriptを始めたばかりの頃は、赤い波線のエラーメッセージに悩まされることが多いですよね。今回は、初心者が特によく遭遇するエラー5選と、その解決策を分かりやすく解説します。

## 1. Property does not exist on type

**エラー例：**
```typescript
const user = { name: "田中", age: 25 };
console.log(user.email); // エラー: Property 'email' does not exist on type
```

**解決策：**
オブジェクトに存在しないプロパティにアクセスしようとした時のエラーです。プロパティ名のタイポを確認するか、型定義を見直しましょう。

```typescript
const user = { name: "田中", age: 25, email: "tanaka@example.com" };
console.log(user.email); // OK
```

## 2. Type 'string' is not assignable to type

**エラー例：**
```typescript
let count: number = "10"; // エラー: Type 'string' is not assignable to type 'number'
```

**解決策：**
型の不一致が原因です。文字列を数値に変換するか、型を統一しましょう。

```typescript
let count: number = 10; // OK
// または
let count: number = parseInt("10"); // OK
```

## 3. Object is possibly 'null' or 'undefined'

**エラー例：**
```typescript
const element = document.getElementById("myButton");
element.addEventListener("click", () => {}); // エラー: Object is possibly 'null'
```

**解決策：**
nullチェックを行うか、非null断言演算子（!）を使用します。

```typescript
const element = document.getElementById("myButton");
if (element) {
  element.addEventListener("click", () => {}); // OK
}
// または
element!.addEventListener("click", () => {}); // OK（要素の存在が確実な場合）
```

## 4. Argument of type is not assignable to parameter

**エラー例：**
```typescript
function greet(name: string) {
  console.log(`Hello, ${name}`);
}

greet(123); // エラー: Argument of type 'number' is not assignable to parameter of type 'string'
```

**解決策：**
関数の引数の型を確認し、正しい型で渡しましょう。

```typescript
greet("太郎"); // OK
// または数値を文字列に変換
greet(String(123)); // OK
```

## 5. Cannot redeclare block-scoped variable

**エラー例：**
```typescript
let message = "Hello";
let message = "World"; // エラー: Cannot redeclare block-scoped variable 'message'
```

**解決策：**
同じ変数名を重複して宣言しています。変数名を変更するか、値の再代入を使いましょう。

```typescript
let message = "Hello";
message = "World"; // OK（再代入）
// または
let message1 = "Hello";
let message2 = "World"; // OK（異なる変数名）
```
