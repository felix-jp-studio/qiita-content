# AWS初心者がまず覚えるべき基本サービス5選

AWS（Amazon Web Services）は200以上のサービスを提供しており、初心者にとってはどこから始めればよいか迷ってしまうことも多いでしょう。今回は、AWS学習の第一歩として押さえておくべき基本的なサービス5つを紹介します。

## 1. EC2（Elastic Compute Cloud）
**仮想サーバーサービス**

EC2は、AWSクラウド上で仮想サーバー（インスタンス）を起動・管理できるサービスです。WebサーバーやAPIサーバーなど、様々な用途で利用されます。

### 主な特徴
- 数分でサーバーを起動可能
- 必要に応じてサーバーのスペックを変更
- 使った分だけの従量課金制

## 2. S3（Simple Storage Service）
**オブジェクトストレージサービス**

S3は、ファイルやデータを保存するためのストレージサービスです。静的Webサイトのホスティングやバックアップ、画像・動画の配信など幅広い用途で活用されています。

```bash
# AWS CLIを使ってS3バケットにファイルをアップロード
aws s3 cp myfile.txt s3://my-bucket-name/

# S3バケット内のファイル一覧を表示
aws s3 ls s3://my-bucket-name/

# ファイルをダウンロード
aws s3 cp s3://my-bucket-name/myfile.txt ./
```

### 主な特徴
- 高い耐久性（99.999999999%）
- 容量制限なし
- Webサイトとして公開も可能

## 3. RDS（Relational Database Service）
**マネージド型データベースサービス**

RDSは、MySQL、PostgreSQL、Oracle、SQL Serverなどのリレーショナルデータベースを簡単に構築・運用できるサービスです。

### 主な特徴
- データベースの管理作業を自動化
- 自動バックアップ機能
- 高可用性構成も簡単に実現

## 4. Lambda
**サーバーレスコンピューティングサービス**

Lambdaは、サーバーの管理なしでコードを実行できるサービスです。イベント駆動型の処理や、マイクロサービスアーキテクチャで活用されます。

### 主な特徴
- サーバー管理不要
- 実行時間分のみ課金
- 様々なプログラミング言語に対応

## 5. CloudWatch
**モニタリング・監視サービス**

CloudWatchは、AWSリソースの監視やログ管理を行うサービスです。システムの健康状態を把握し、異常時にアラートを受け取ることができます。

### 主な特徴
- リアルタイム監視
- カスタムメトリクスの設定
- ログの集約・分析

## 学習の進め方

1. **まずは無料利用枠を活用**
   AWSの無料利用枠（Free Tier）を使って実際に触ってみましょう。

2. **小さなプロジェクトから始める**
   静的サイトのS3ホスティングや、簡単なWebアプリケーションの構築から始めることをおすすめします。

3. **公式ドキュメントを活用**
   AWSの公式ドキュメントは非常に充実しているので、積極的に参照しましょう。

## まとめ

今回紹介した5つのサービスは、AWSの基盤となる重要なサービスです。これらを理解することで、より複雑なAWSアーキテクチャも理解しやすくなります。まずは無料利用枠を使って実際に触れてみることから始めてみてください。
