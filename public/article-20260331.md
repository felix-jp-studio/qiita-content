# はじめに

プログラミングを始めたばかりの方にとって、作成したWebアプリケーションを公開する作業は複雑に感じられるかもしれません。しかし、現在では**GitHubにコードをpushするだけで自動的にデプロイできる**仕組みが簡単に構築できます。

この記事では、初心者の方でも理解しやすいように、自動デプロイの基本概念から実装方法まで解説します。

## 自動デプロイとは？

自動デプロイとは、コードをリポジトリにpushした際に、自動的にWebサイトやアプリケーションが更新・公開される仕組みのことです。

**従来の手動デプロイ**
1. コードを修正
2. ファイルをサーバーにアップロード
3. 設定ファイルを更新
4. サーバーを再起動

**自動デプロ���**
1. コードを修正
2. GitHubにpush → **自動的に本番環境に反映**

## Vercelを使った自動デプロイの設定

今回は**Vercel**というプラットフォームを使って、React/Next.jsアプリケーションの自動デプロイを実現します。

### 1. GitHubリポジトリの準備

まず、デプロイしたいプロジェクトをGitHubにpushしておきます。

```bash
# プロジェクトをGitで管理
git init
git add .
git commit -m "Initial commit"

# GitHubリポジトリにpush
git remote add origin https://github.com/yourusername/your-repo.git
git branch -M main
git push -u origin main
```

### 2. Vercelとの連携設定

1. [Vercel](https://vercel.com)にアクセスしてGitHubアカウントでログイン
2. 「New Project」をクリック
3. GitHubリポジトリを選択
4. 「Deploy」をクリック

これだけで基本的な自動デプロイが完成します！

### 3. 環境変数の設定

本番環境で使用する環境変数がある場合は、Vercelの管理画面で設定します。

```javascript
// 例：APIキーを環境変数として使用
const apiKey = process.env.NEXT_PUBLIC_API_KEY;

fetch(`https://api.example.com/data?key=${apiKey}`)
  .then(response => response.json())
  .then(data => console.log(data));
```

## GitHub Actionsを使った自動デプロイ

より高度な自動デプロイを実現したい場合は、**GitHub Actions**を使用できます。

```yaml
# .github/workflows/deploy.yml
name: Deploy to Production

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v2
    
    - name: Setup Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '18'
    
    - name: Install dependencies
      run: npm install
    
    - name: Build application
      run: npm run build
    
    - name: Deploy to Vercel
      uses: amondnet/vercel-action@v20
      with:
        vercel-token: ${{ secrets.VERCEL_TOKEN }}
        vercel-org-id: ${{ secrets.ORG_ID }}
        vercel-project-id: ${{ secrets.PROJECT_ID }}
```

## 自動デプロイのメリット

### 1. 作業効率の向上
手動でのファイルアップロードや設定変更が不要になり、開発により集中できます。

### 2. ヒューマンエラーの削減
自動化により、デプロイ時の設定ミスや手順の抜け漏れを防げます。

### 3. 迅速なリリース
修正や新機能の追加を即座に本番環境に反映できます。

## 注意点

- **main/masterブランチ**への直接pushは慎重に行う
- **テスト**を自動化プロセスに含めることを推奨
- **ロールバック**の方法を事前に確認しておく
