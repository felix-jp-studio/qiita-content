# GitHubにpushするだけで自動デプロイを実現する方法

プログラミング初心者の皆さん、コードを書いた後にWebサイトを公開する作業が面倒だと感じていませんか？今回は、GitHubにコードをpushするだけで自動的にWebサイトがデプロイされる仕組みを構築する方法を解説します。

## 自動デプロイとは？

自動デプロイとは、コードの変更を検知して自動的にWebサイトやアプリケーションを本番環境に反映させる仕組みのことです。手動でのアップロード作業が不要になり、開発効率が大幅に向上します。

## 今回使用するツール

- **GitHub**: コード管理
- **Vercel**: デプロイプラットフォーム
- **React/Next.js**: Webアプリケーションフレームワーク（例として）

## 基本的な流れ

1. GitHubリポジトリにコードをpush
2. Vercelが変更を自動検知
3. 自動的にビルド・デプロイが実行
4. 新しいバージョンが公開される

## 実装手順

### 1. GitHubリポジトリの準備

まず、デプロイしたいプロジェクトをGitHubリポジトリにpushしておきます。

```bash
# プロジェクトディレクトリで実行
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/username/repository-name.git
git push -u origin main
```

### 2. Vercelとの連携設定

1. [Vercel](https://vercel.com/)にアクセスしてGitHubアカウントでログイン
2. 「New Project」をクリック
3. デプロイしたいGitHubリポジトリを選択
4. 必要に応じて設定を調整して「Deploy」をクリック

### 3. 自動デプロイの確認

設定完了後、GitHubにコードをpushするたびに以下のプロセスが自動実行されます：

- ソースコードの取得
- 依存関係のインストール
- ビルドの実行
- 本番環境への反映

## package.jsonの設定例

Vercelで自動デプロイを行う場合、以下のようなscriptsが設定されていることを確認してください：

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "scripts": {
    "build": "next build",
    "start": "next start",
    "dev": "next dev"
  },
  "dependencies": {
    "next": "^13.0.0",
    "react": "^18.0.0",
    "react-dom": "^18.0.0"
  }
}
```

## メリット

- **効率性**: 手動アップロードが不要
- **一貫性**: 毎回同じプロセスでデプロイされる
- **迅速性**: pushから公開まで数分で完了
- **履歴管理**: デプロイ履歴が自動で記録される

## 注意点

- **環境変数**: 本番環境で必要な環境変数はVercelの設定画面で登録
- **ビルドエラー**: エラーがある場合はデプロイが失敗するため、事前にローカルでテストを実行
- **ブランチ設定**: デフォルトではmainブランチがデプロイ対象

## まとめ

GitHubとVercelを連携させることで、コードをpushするだけで自動デプロイが実現できます。これにより開発効率が大幅に向上し、より多くの時間をコーディングに費やすことができるようになります。

初心者の方も簡単に設定できるので、ぜひ試してみてください。一度設定してしまえば、その後の開発がとてもスムーズになります。
