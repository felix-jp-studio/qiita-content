# 初心者がつまずくTypeScriptのエラー5選と解決策

TypeScriptを始めたばかりの頃は、JavaScriptでは見慣れないエラーに遭遇して困惑することが多いでしょう。今回は、初心者がよく遭遇する5つのTypeScriptエラーと、その解決策をわかりやすく解説します。

## 1. Type 'string | undefined' is not assignable to type 'string'

これは、値がundefinedの可能性があるのに、string型として扱おうとした時に発生するエラーです。

```typescript
// エラーが発生するコード
const user = {
  name: "田中",
  email: "tanaka@example.com"
};

const userName: string = user.name; // OK
const userAge: string = user.age;   // エラー！

// 解決策1: オプショナルチェーンとデフォルト値
const userAge: string = user.age || "未設定";

// 解決策2: 型ガードを��用
if (user.age !== undefined) {
  const userAge: string = user.age;
}
```

## 2. Object is possibly 'null'

オブジェクトがnullの可能性があるのに、プロパティにアクセスしようとした時のエラーです。

```typescript
// エラーが発生するコード
const element = document.getElementById("myButton");
element.addEventListener("click", () => {}); // エラー！

// 解決策: null チェック
const element = document.getElementById("myButton");
if (element) {
  element.addEventListener("click", () => {});
}

// または非null アサーション演算子を使用（要注意）
const element = document.getElementById("myButton")!;
```

## 3. Argument of type 'string' is not assignable to parameter of type 'number'

関数の引数の型が一致しない時のエラーです。

```typescript
function calculateArea(width: number, height: number): number {
  return width * height;
}

// エラーが発生するコード
const area = calculateArea("10", "20"); // エラー！

// 解決策: 型変換
const area = calculateArea(Number("10"), Number("20"));
// または
const area = calculateArea(parseInt("10"), parseInt("20"));
```

## 4. Property 'xxx' does not exist on type

オブジェクトに存在しないプロパティにアクセスしようとした時のエラーです。

```typescript
interface User {
  name: string;
  email: string;
}

const user: User = {
  name: "山田",
  email: "yamada@example.com"
};

console.log(user.age); // エラー！

// 解決策1: インターフェースにプロパティを追加
interface User {
  name: string;
  email: string;
  age?: number; // オプショナルプロパティとして追加
}

// 解決策2: 型アサーション（推奨されない）
console.log((user as any).age);
```

## 5. Cannot find name 'xxx'

変数や関数が定義されていない時のエラーです。

```typescript
// エラーが発生するコード
console.log(userName); // エラー！

function greet() {
  console.log(`Hello, ${userName}!`); // エラー！
}

// 解決策: 適切に変数を定義
const userName: string = "佐藤";

function greet() {
  console.log(`Hello, ${userName}!`); // OK
}
```

## まとめ

これらのエラーは、TypeScriptの型安全性を保つための重要な機能です。最初は面倒に感じるかもしれませんが、慣れてくるとバグを未然に防いでくれる頼もしい存在になります。

エラーメッセージを恐れずに、一つずつ丁寧に解決していけば、必ずTypeScriptを使いこなせるようになります。エラーが出たら、まずはエラーメッセージをよく読んで、何が問題なのかを理解することから始めましょう。
