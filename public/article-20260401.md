# 初心者がつまずくTypeScriptのエラー5選と解決策

TypeScriptを始めたばかりの頃、JavaScriptとの違いに戸惑い、様々なエラーに遭遇することがあります。今回は、初心者が特によく遭遇する5つのエラーパターンと、その解決策を紹介します。

## 1. Type 'string' is not assignable to type 'number'

最も基本的な型エラーです。文字列を数値型の変数に代入しようとした際に発生します。

```typescript
// ❌ エラーになるコード
let age: number = "25";

// ✅ 正しいコード
let age: number = 25;
// または
let age: number = parseInt("25");
```

**解決策**: 変数の型と代入する値の型を一致させるか、適切な型変換を行いましょう。

## 2. Object is possibly 'null' or 'undefined'

nullやundefinedの可能性がある値にアクセスしようとした際のエ��ーです。

```typescript
// ❌ エラーになるコード
function getUserName(user: { name: string } | null) {
  return user.name; // userがnullの可能性がある
}

// ✅ 正しいコード
function getUserName(user: { name: string } | null) {
  if (user) {
    return user.name;
  }
  return "Unknown";
}

// またはオプショナルチェーニングを使用
function getUserName(user: { name: string } | null) {
  return user?.name || "Unknown";
}
```

**解決策**: null/undefinedチェックを行うか、オプショナルチェーニング（?.）を活用しましょう。

## 3. Property 'xxx' does not exist on type

存在しないプロパティにアクセスしようとした際のエラーです。

```typescript
// ❌ エラーになるコード
interface User {
  name: string;
  age: number;
}

const user: User = { name: "田中", age: 30 };
console.log(user.email); // emailプロパティは存在しない

// ✅ 正しいコード
interface User {
  name: string;
  age: number;
  email?: string; // オプショナルプロパティとして追加
}

const user: User = { name: "田中", age: 30 };
console.log(user.email); // undefinedが返される
```

**解決策**: インターフェースにプロパティを追加するか、オプショナルプロパ��ィ（?）を使用しましょう。

## 4. Argument of type 'string' is not assignable to parameter of type 'never'

Union型の絞り込みが不十分な場合に発生するエラーです。

```typescript
// ❌ エラーになるコード
function processValue(value: string | number) {
  if (typeof value !== 'string' && typeof value !== 'number') {
    // この条件分岐は到達不可能
    console.log(value); // valueの型がneverになる
  }
}

// ✅ 正しいコード
function processValue(value: string | number) {
  if (typeof value === 'string') {
    console.log(`文字列: ${value}`);
  } else {
    console.log(`数値: ${value}`);
  }
}
```

**解決策**: 型ガードを正しく使用し、すべての可能性のある型を適切に処理しましょう。

## 5. Cannot find module 'xxx' or its corresponding type declarations

モジュールの型定義が見つからない際のエラーです。

```typescript
// ❌ エラーになるコード（lodashの型定義がない場合）
import _ from 'lodash'; // 型定義が見つからない

// ✅ 解決方法
// 1. 型定義をインストール
// npm install --save-dev @types/lodash

// 2. または独自の型定義を作成
declare module 'lodash' {
  export function isEmpty(value: any): boolean;
  // 必要な関数のみ定義
}
```

**解決策**: `@types/パッケージ名`で型定義をインストールするか、独自の型定義を作成しましょう。
