# GitHubにpushするだけで自動デプロイを実現する方法

プログラミング初心者の皆さん、毎回手動でアプリをデプロイするのって面倒ですよね？今回は、GitHubにコードをpushするだけで自動的にアプリがデプロイされる仕組み（CI/CD）の作り方を解説します。

## CI/CDとは？

CI/CD（Continuous Integration/Continuous Deployment）は、コードの変更を自動的にテスト・デプロイする仕組みです。開発者がコードをpushすると、以下の流れが自動で実行されます：

1. コードの品質チェック
2. テストの実行
3. アプリケーションのビルド
4. 本番環境へのデプロイ

## GitHub Actionsを使った自動デプロイの設定

GitHub Actionsは、GitHubが提供する無料のCI/CDサービスです。リポジトリ内に設定ファイルを置く��けで簡単に自動化できます。

### 1. ワークフローファイルの作成

プロジェクトのルートに `.github/workflows/deploy.yml` ファイルを作成します：

```yaml
name: Deploy to Production

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
      
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
        
    - name: Install dependencies
      run: npm ci
      
    - name: Run tests
      run: npm test
      
    - name: Build application
      run: npm run build
      
    - name: Deploy to Vercel
      uses: amondnet/vercel-action@v25
      with:
        vercel-token: ${{ secrets.VERCEL_TOKEN }}
        vercel-org-id: ${{ secrets.ORG_ID }}
        vercel-project-id: ${{ secrets.PROJECT_ID }}
        vercel-args: '--prod'
```

### 2. 環境変数の設定

GitHubリポジトリの設定で、以下の秘密情報を登録します：

- `VERCEL_TOKEN`: Vercelのアクセストークン
- `ORG_ID`: VercelのOrganization ID
- `PROJECT_ID`: VercelのProject ID

Settings > Secrets and variables > Actions から追加できます。

## メリット

1. **時間短縮**: 手動デプロイ���時間を削減
2. **ヒューマンエラーの防止**: 手動作業によるミスを回避
3. **品質向上**: 自動テストにより品質を保証
4. **チーム開発の効率化**: メンバー全員が同じプロセスでデプロイ可能

## 注意点

- 初期設定には時間がかかる場合があります
- テストが失敗するとデプロイが止まるため、テストコードの品質も重要
- 秘密情報の管理は慎重に行いましょう

## まとめ

GitHub Actionsを使った自動デプロイは、一度設定すれば開発効率が大幅に向上します。最初は複雑に感じるかもしれませんが、基本的な設定から始めて徐々に機能を追加していけば大丈夫です。
