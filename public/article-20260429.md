TypeScriptを始めたばかりの頃、エラーメッセージに戸惑うことはありませんか？今回は、初心者が特によく遭遇する5つのエラーパターンとその解決策を紹介します。

## 1. Property '○○' does not exist on type

最も頻出するエラーの一つです。オブジェクトのプロパティが存在しないか、型定義が不適切な場合に発生します。

```typescript
// エラーが発生するコード
const user = {
  name: "田中",
  age: 25
};
console.log(user.email); // エラー！

// 解決策1: プロパティを追加
const user = {
  name: "田中",
  age: 25,
  email: "tanaka@example.com"
};

// 解決策2: 型定義を使用
interface User {
  name: string;
  age: number;
  email?: string; // オプショナルプロパティ
}
```

## 2. Argument of type '○○' is not assignable to parameter of type

関数の引数に渡す値の型が期待される型と一致しない場合のエラーです。

```typescript
// エラーが発生するコード
function greet(name: string) {
  return `Hello, ${name}!`;
}
greet(123); // エラー！

// 解決策: 正しい型で呼び出す
greet("田中");

// または型変換
greet(String(123));
```

## 3. Cannot find module '○○' or its corresponding type declarations

モジュールのインポート時によく見るエラーです。主にライブラリの型定義が不足している場合に発生します。

```bash
# 解決策: 型定義をインストール
npm install @types/node
npm install @types/react
npm install @types/lodash
```

## 4. Object is possibly 'undefined'

strictモードで頻出するエラーです。変数がundefinedの可能性がある場合に発生します。

```typescript
// エラーが発生するコード
const users = [
  { name: "田中", age: 25 },
  { name: "佐藤", age: 30 }
];
const firstUser = users.find(user => user.age > 20);
console.log(firstUser.name); // エラー！

// 解決策1: 条件分岐で対処
if (firstUser) {
  console.log(firstUser.name);
}

// 解決策2: オプショナルチェーニング
console.log(firstUser?.name);

// 解決策3: Non-null assertion operator（確実な場合��み）
console.log(firstUser!.name);
```

## 5. Type 'string' is not assignable to type 'never'

配列やユニオン型で型推論が正しく働かない場合に発生することがあります。

```typescript
// エラーが発生するコード
const items = [];
items.push("hello"); // エラーの可能性

// 解決策: 型注釈を追加
const items: string[] = [];
items.push("hello"); // OK

// または初期値を設定
const items = [""];
items.push("hello"); // OK
```

## エラー解決のコツ

1. **エラーメッセージをしっかり読む**: TypeScriptのエラーメッセージは比較的親切です
2. **型定義を確認する**: `hover`やVSCodeの機能を活用して型を確認しましょう
3. **段階的に型を追加**: 最初から完璧を目指さず、徐々に型安全性を高めていきましょう
