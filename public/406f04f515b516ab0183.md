# AWS初心者がまず覚えるべきサービス5選

AWSには200以上のサービスがありますが、初心者の方はどれから学べばいいか迷ってしまいますよね。今回は、AWS初心者が最初に覚えるべき5つのコアサービスをご紹介します。

## 1. EC2（Elastic Compute Cloud）

EC2はAWSの仮想サーバーサービスです。物理サーバーを購入することなく、クラウド上で簡単にサーバーを立ち上げられます。

**主な特徴**
- 数分でサーバーを起動可能
- 使った分だけの従量課金
- 様々なOSやスペックを選択可能

Webアプリケーションの公開やデータ処理など、幅広い用途で使用されます。

## 2. S3（Simple Storage Service）

S3はAWSのオブジェクトストレージサービスです。写真、動画、ドキュメントなど、あらゆる種類のファイルを安全に保存できます。

**主な特徴**
- 99.999999999%（イレブンナイン）の耐久性
- 静的ウェブサイトのホスティングが可能
- 容量制限なし

```bash
# AWS CLIを使ってファイルをS3にアップロード
aws s3 cp myfile.txt s3://my-bucket/myfile.txt

# S3バケットの内容を確認
aws s3 ls s3://my-bucket/
```

## 3. RDS（Relational Database Service）

RDSはマネージドなリレーショナルデータベースサービスです。MySQL、PostgreSQL、Oracle、SQL Serverなど、主要なデータベースエンジンに対応しています。

**主な特徴**
- 自動バックアップ機能
- メンテナンスの自動化
- 高可用性構成が簡単に構築可能

データベースの運用管理をAWSが代行してくれるため、開発に集中できます。

## 4. Lambda

Lambdaはサーバーレスコンピューティングサービスです。サーバーの管理なしに、コードを実行できます。

**主な特徴**
- サーバー管理不要
- 実行時間に応じた課金
- イベント駆動での自動実行

画像のリサイズ、データの加工、APIの作成など、様々な処理を効率的に実行できます。

## 5. IAM（Identity and Access Management）

IAMはAWSリソースへのアクセスを安全に管理するサービスです。ユーザー、���ループ、ロールを作成し、きめ細かい権限設定が可能です。

**主な特徴**
- ユーザーごとの権限制御
- 多要素認証（MFA）対応
- 一時的なアクセス権限の付与

セキュリティの観点から、AWSを使う上で必須のサービスです。

## まとめ

これら5つのサービスを理解することで、AWSの基本的なアーキテクチャが組めるようになります。

1. **EC2**でアプリケーションサーバーを構築
2. **S3**で静的ファイルを保存
3. **RDS**でデータベースを管理
4. **Lambda**で軽量な処理を実行
5. **IAM**でセキュリティを確保
