# AWS初心者がまず覚えるべきサービス5選

AWSには200を超えるサービスがありますが、初心者の方は「何から始めればいいの？」と迷ってしまいますよね。この記事では、プログラミング初心者が最初に覚えるべきAWSサービスを5つに絞って紹介します。

## 1. EC2（Elastic Compute Cloud）

**EC2は仮想サーバーを提供するサービス**です。従来の物理サーバーを借りる代わりに、クラウド上でサーバーを立てることができます。

### できること
- Webアプリケーションのホスティング
- 開発・テスト環境の構築
- バッチ処理の実行

### 料金体系
使った分だけ課金される従量課金制で、無料枠では月750時間まで利用できます。

## 2. S3（Simple Storage Service）

**S3��オブジェクトストレージサービス**で、ファイルやデータを安全に保存できます。

### 使用例
```bash
# AWS CLIでS3にファイルをアップロード
aws s3 cp myfile.txt s3://my-bucket/myfile.txt

# S3からファイルをダウンロード
aws s3 cp s3://my-bucket/myfile.txt ./downloaded-file.txt
```

### 特徴
- 99.999999999%（イレブンナイン）の耐久性
- 静的Webサイトのホスティングも可能
- 画像や動画などのメディアファイル保存に最適

## 3. RDS（Relational Database Service）

**RDSは管理型データベースサービス**です。MySQL、PostgreSQL、Oracleなどの主要データベースエンジンに対応しています。

### メリット
- データベースの運用管理が自動化
- バックアップ・復旧が簡単
- スケーリングが容易

### 初心者におすすめする理由
データベースサーバーの構築や保守を自分で行う必要がないため、アプリケーション開発に集中できます。

## 4. CloudFront

**CloudFrontはCDN（Content Delivery Network）サービス**です。世界中にあるエッジロケーションからコンテンツを配信することで、ユーザーに高速でコンテンツを届けます。

### 効果
- Webサイトの表示速度向��
- サーバー負荷の軽減
- グローバルなユーザー体験の向上

特にS3と組み合わせて静的サイトを配信する際によく使われます。

## 5. IAM（Identity and Access Management）

**IAMはAWSリソースへのアクセスを制御するサービス**です。セキュリティの要となる重要なサービスです。

### 主な機能
- ユーザー・グループ・ロールの管理
- アクセス権限の細かい設定
- 多要素認証（MFA）の設定

### セキュリティのベストプラクティス
- ルートアカウントは使わない
- 最小権限の原則を守る
- 定期的にアクセスキーを更新する

## まとめ

この5つのサービスを理解することで、AWSでの基本的なWebアプリケーション構築ができるようになります。

**学習の順序としては：**
1. IAMでセキュリティ設定
2. EC2でサーバー構築
3. RDSでデータベース準備
4. S3でファイル保存
5. CloudFrontで高速配信

実際に手を動かして試してみることが一番の学習方法です。AWS無料枠を活用して、まずは小さなプロジェクトから始めてみましょう。
